Octal Engine Changelog

Current version: 0.1.0-beta
First version: 0.0.1

0.1.0-beta

- Added new third-party library dependency: bgfx.
- Added build_bgfx.ps1 script to automate bgfx build and integration.
- Integrated bgfx into the project with CMake support.
- Added OctalEngineJobs target with a lightweight thread-pool job system.
- Added OctalEngineRenderer target with a public Renderer facade.
- Added RendererInitSettings with headless and windowed rendering support.
- Added Mesh, Vertex, VertexBuffer, IndexBuffer, Material stub, Mat4, and
  RenderCommand public renderer-facing types.
- Added internal render command buffers and a dedicated render thread.
- Isolated bgfx usage inside engine/renderer/backend.
- Added bgfx-backed mesh upload, shader loading, frame submission, and resize
  handling.
- Added third_party/bgfx/shaders cube shader assets used by the minimal backend.
- Added renderer headless mode that accepts render commands without bgfx/GPU
  initialization.
- Added Renderer::resize(width, height).
- Added native OS window handle lookup for Window::nativeHandle().
- Added per-window event subscriptions through window->events().
- Added window resize, move, minimize, maximize, restore, show, hide, focus,
  and close-request events.
- Routed SDL window events to the specific Window instance that owns them.
- Updated TestSandbox to create a window, initialize Renderer, submit a rotating
  cube from engine-driven Update events, and resize the renderer from that
  window's resize event.
- Updated public docs for renderer, job system, per-window events, and the cube
  rendering example.
- Updated OctalEngine_AddGame to link renderer and jobs targets when available.
- Aligned MSVC runtime settings with the bundled bgfx static libraries.
- Added InputManager with action-based input polling and callbacks.
- Added input contexts with priority and blocking behavior.
- Added runtime rebinding, save/load binding stubs, and default binding reset.
- Added raw keyboard, mouse, gamepad, cursor, and touch-facing API.
- Wired SDL keyboard and mouse events into the input system through
  PlatformSystem.
- Advanced input frame state from GameLoop::update().
- Updated TestSandbox to showcase input actions, context setup, runtime
  rebinding, Escape-to-quit, and renderer resize from window events.

0.1.0-alpha

- Added public user documentation under docs/public.
- Made docs/public/README.md the main public documentation navigation page.
- Documented getting started, engine loop concepts, and public API references.
- Added API docs for Engine, EngineConfig, Platform, PlatformSystem, Window,
  Time, and GameLoop.
- Added a platform abstraction layer so the engine can integrate platform events
  without requiring windowing.
- Added Platform as the core event-pumping interface.
- Added PlatformSystem as the public platform/window manager.
- Added Window, WindowDescriptor, WindowSize, WindowPosition, and WindowId APIs.
- Added window creation, lookup, close, close-all, and window counting support.
- Added runtime window controls for title, size, position, centering,
  visibility, maximize, minimize, restore, fullscreen, resizable state, and
  native handle access.
- Integrated platform event pumping into the engine main loop.
- Kept headless engine usage supported: engine.run() still works without a
  window or platform target.
- Kept the engine core independent from the platform/windowing target.
- Added an optional OctalEnginePlatform CMake target.
- Added OCTAL_BUILD_PLATFORM CMake option to build with or without the platform
  layer.
- Added CMake integration for the internal SDL2 backend while keeping SDL2 out
  of public headers, user samples, and public documentation.
- Made SDL2 a private implementation detail of OctalEnginePlatform.
- Added SDL2 DLL copy step for the windowed sandbox on Windows.
- Refactored bootstrap API from setPlatform() to constructor injection.
- Removed Engine::setPlatform().
- Added EngineConfig using std::variant for runtime mode configuration.
- Added HeadlessMode for simulations, tests, tools, and server-style workloads
  that do not need a window.
- Added WindowedMode with Window* window for windowed runtime setup.
- Added Engine constructors:
  - Engine()
  - Engine(const EngineConfig&)
  - Engine(Platform&, const EngineConfig&)
- Avoided public/user-code preprocessor branching for platform support.
- Split sandbox samples into:
  - test/main.cpp for headless bootstrap
  - test/windowed.cpp for windowed bootstrap
- Added WindowedSandbox executable when the platform layer is enabled.
- Updated samples to avoid designated initializers so editor IntelliSense stays
  clean.
- Verified platform-enabled and headless CMake builds.
- Added Octal Engine CMake package config support for find_package.
- Added OctalEngine_AddGame alias for the helper.
- Hid engine target linking, C++ standard setup, and runtime backend file copy
  behind the game helper.
- Added CMake install/export rules for engine targets, headers, package config,
  version file, and helper module.
- Added build-tree package files so consumers can test find_package against the
  local build directory.
- Added typed event system with EventWorld, EventBus, and Subscription.
- Added RAII event subscriptions that auto-unsubscribe when destroyed.
- Added once listeners that remove themselves after the first matching event.
- Added immediate emit() support.
- Added deferred emitDeferred() support with EventWorld::flush().
- Added built-in Update event with dt.
- Engine now emits Update events on the engine bus each frame and flushes
  deferred events after the frame's placeholder render step.

0.0.1

- Initial Octal Engine project version.
- Added basic C++20 CMake project structure.
- Added OctalEngine static library target.
- Added Sandbox executable target.
- Added initial engine core classes:
  - Engine
  - Time
  - GameLoop
- Added a simple Engine::run() loop that steps time, updates, and renders.
- Added Engine::stop().
- Added fixed timestep placeholder returning 1.0f / 60.0f.
- Added initial repository documentation and internal project notes.
