# Information about project internals

For a better understanding of the project structure and its components and AIs to navigate through the codebase.

**Project plans**: [See here](/PLAN.md)

---

`build/` - contains the build output of the project. This folder is generated by the build process and should not be modified manually and ignored in version control.

`test/` - contains test files for the project. This folder is used to organize and store tests that ensure the functionality and correctness of the codebase.

`test/main.cpp` - the main entry point for running tests. This file typically includes the necessary headers and sets up the testing framework to execute the tests defined in the `test/` folder.

`engine/` - contains the core engine code of the project. This folder is where the main logic and functionality of the project are implemented.

`engine/core/` - contains the core components of the engine, such as the main application loop.

---

The generated `Sandbox` executable is the result of building the project. It serves as a sandbox environment for testing and experimenting with the engine's features and functionalities.

And `OctalEngine.lib` is the static library that contains the compiled code of the engine. It can be linked to other projects to utilize the engine's functionality without needing to include the source code directly.

---

Engine owns the main loop and calls other layers to update and render the game. The main loop is responsible for managing the flow of the game, including handling input, updating game logic, and rendering graphics. It ensures that the game runs smoothly and efficiently by controlling the timing and execution of these tasks.
